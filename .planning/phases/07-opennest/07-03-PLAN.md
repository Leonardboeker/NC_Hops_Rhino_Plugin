  <action>
Human verifies the full nesting workflow in a live Grasshopper session.

**What was built:** HopPart component (bundles part outline + operationLines into Dictionary) and HopSheetExport component (filters parts by OpenNest IDS sheet index, writes per-sheet .hop file with identical structure to HopExport).

**Prerequisites:** OpenNest 2.1.0 installed via Rhino 8 Package Manager.

**Test Setup in GH (build from scratch or extend existing definition):**

1. Create 2-3 simple rectangular closed curves in Rhino (e.g. 200x300mm, 150x250mm, 180x220mm) -- these are the part outlines
2. For each part, wire the outline through a HopContour component (or any op-component) to get operationLines
3. Wire each part's outline + operationLines into a HopPart component
4. Create a sheet rectangle (e.g. 800x400mm closed polyline) for the nesting boundary
5. Wire the HopPart outlines (extract from HopPart or use original curves) into OpenNest Geo input
6. Wire the sheet rectangle into OpenNest Sheets input
7. Set OpenNest: Spacing=2, Rotations=4, Run=true
8. After OpenNest solves: apply the Transform output to the original detail geometry using GH Transform component
9. Re-wire the transformed geometry through op-components to get nested-position operationLines
10. Wire HopPart outputs + OpenNest IDS into HopSheetExport
11. Wire the sheet curve from OpenNest Sheets output into HopSheetExport sheetCurve
12. Set sheetIndex=0, folder to a test directory, fileName to "test_sheet_0"
13. Toggle export=true

**Verify each of these:**

a. HopPart preview: Each HopPart shows the outline curve + a grain direction arrow in the viewport
b. OpenNest nesting: Parts are nested on the sheet(s) -- visible in viewport
c. HopSheetExport output: File test_sheet_0.hop is created in the specified folder
d. File content check: Open the .hop file in a text editor and verify header starts with ;MAKROTYP=0, contains VARS block with correct DX/DY matching the sheet dimensions, contains START + Fertigteil + CALL HH_Park, and operation lines have XY coordinates matching nested positions (NOT original positions)
e. Multi-sheet test (if parts overflow): Wire a second HopSheetExport with sheetIndex=1 to verify multi-sheet export
f. Error handling: Disconnect the outline from HopPart -- should show Error
g. Unfitted warning: If any IDS=-1 parts exist, HopSheetExport should show a Warning

Resume signal: Type "approved" if the nesting workflow works correctly and .hop files contain correct nested-position coordinates. Otherwise describe any issues.
