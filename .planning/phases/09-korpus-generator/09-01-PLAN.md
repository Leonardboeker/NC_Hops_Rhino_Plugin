Create the HopKorpus Grasshopper component that takes 4 numeric sliders (B, H, T, Materialstaerke) and a text label (Korpus-Typ) and generates 6 flat panel Breps representing a cabinet body. All 5 inputs marked Optional = true (they all have defaults).

**RegisterOutputParams (2 outputs):**
1. `AddGenericParameter("KorpusData", "korpus", "Complete korpus dictionary wrapped in GH_ObjectWrapper. Contains B, H, T, thickness, type, and panel list. Wire into downstream Korpus components (HopKorpusRueckwand, HopVerbinder, etc.).", GH_ParamAccess.item)`
2. `AddGenericParameter("Panels", "panels", "Individual flat panel dictionaries (GH_ObjectWrapper), one per panel. Each contains outline curve, operationLines, grainDir, panelName. Wire into HopPart for nesting.", GH_ParamAccess.list)`

**SolveInstance logic:**

1. Get all 5 inputs with defaults:
   - `double B = 600, H = 720, T = 560, MS = 19; string korpusTyp = "Korpus";`
   - Use `DA.GetData(0, ref B)` etc. for each.

2. Guards:
   - B <= 0, H <= 0, T <= 0: `AddRuntimeMessage(Error, "Dimensions must be > 0")`, return
   - MS <= 0 or MS >= Math.Min(B, Math.Min(H, T)) / 2: `AddRuntimeMessage(Error, "Materialstaerke invalid")`, return

3. Calculate inner dimensions:
   - `double innerB = B - 2 * MS;` (inner width, between side panels)
   - `double innerH = H - 2 * MS;` (inner height, between Boden and Deckel)
   - `double innerT = T;` (depth stays same for butt-joint construction)

4. Create 5 KorpusPanel objects (NO front panel -- open front is an absence):

   **Boden (bottom):**
   - `new KorpusPanel("Boden", innerB, T, MS)` — width = innerB (sits BETWEEN side panels in butt joint), height = T (full depth)

   **Deckel (top):**
   - `new KorpusPanel("Deckel", innerB, T, MS)` — same dimensions as Boden

   **LinkeSeite (left side):**
   - `new KorpusPanel("LinkeSeite", T, H, MS)` — width = T (depth), height = H (full height, sandwiches Boden/Deckel)

   **RechteSeite (right side):**
   - `new KorpusPanel("RechteSeite", T, H, MS)` — same as LinkeSeite

   **Rueckwand (back wall, placeholder):**
   - `new KorpusPanel("Rueckwand", innerB, innerH, MS)` — width = innerB, height = innerH (fits inside all 4 panels). Phase 9.1 placeholder: full-back panel using same MS thickness. Phase 9.2 will replace with actual Rueckwand options component.

5. Compute assembled transforms for each panel (for 3D preview in Plan 09-02):
   The assembled korpus has its front-bottom-left corner at world origin (0,0,0).
   Butt joint construction: side panels run full height, Boden/Deckel sit between sides.

   **Boden:** translate flat Brep so it sits at position (MS, 0, 0), rotated to lie in XY plane at Z=0. Actually flat panel is already XY. For assembled position: move to (MS, 0, 0) — the Boden is horizontal, its flat XY matches the assembled XY. No rotation needed.
   - `AssembledTransform = Transform.Translation(MS, 0, 0)`

   **Deckel:** same as Boden but at Z = H - MS.
   - `AssembledTransform = Transform.Translation(MS, 0, H - MS)`

   **LinkeSeite:** flat panel has width=T, height=H at XY. In assembled position it stands vertically at X=0, rotated 90 degrees around Y axis... Actually, let me think about this correctly:

   The flat panel lies at Z=0 with width along X and height along Y. For assembled position:
   - LinkeSeite: stands vertically on the LEFT side. In the assembled korpus, it occupies the YZ plane at X=0. The panel's flat width (T) maps to Y (depth), and flat height (H) maps to Z (height). So we need a transform that maps: X->Y, Y->Z. This is a rotation of 90 degrees around X-axis, then no further rotation. Actually:

   Simpler approach -- compute the assembled transform directly:
   - Create a `Transform` that maps the flat panel (at XY, Z=0) to its assembled 3D position.

   For **LinkeSeite** (left, stands at X=0 in YZ plane):
   Flat: corners (0,0,0)-(T,0,0)-(T,H,0)-(0,H,0)
   Assembled: corners (0,0,0)-(0,T,0)-(0,T,H)-(0,0,H)
   Transform: rotate 90 deg around Z-axis (X->Y), then rotate 90 deg around X-axis (Y->Z).
   Or use: `Transform.Rotation(Math.PI/2, Vector3d.ZAxis, Point3d.Origin)` then `Transform.Rotation(Math.PI/2, Vector3d.XAxis, Point3d.Origin)`.

   Actually, simplify by building transforms via `Transform.PlaneToPlane`:
   - Source plane: `Plane.WorldXY` (flat panel lives here)
   - Target plane: the plane where the panel sits in the assembled korpus

   **LinkeSeite target plane:** Origin=(0,0,0), XAxis=(0,1,0) (width=T maps to Y), YAxis=(0,0,1) (height=H maps to Z)
   → `new Plane(new Point3d(0,0,0), new Vector3d(0,1,0), new Vector3d(0,0,1))`

   **RechteSeite target plane:** Origin=(B,0,0), XAxis=(0,1,0), YAxis=(0,0,1)  — but panel faces inward, so XAxis=(0,1,0) is fine (mirrored is cosmetic, not structural)
   → `new Plane(new Point3d(B-MS,0,0), new Vector3d(0,1,0), new Vector3d(0,0,1))`

   **Boden target plane:** Origin=(MS,0,0), XAxis=(1,0,0), YAxis=(0,1,0) — horizontal, lying in XY
   Actually Boden flat panel has width=innerB along X, height=T along Y. In assembled position: X maps to X (shifted by MS), Y maps to Y. Already XY plane.
   → `Transform.Translation(MS, 0, 0)`
   Simpler: just translate. Use `Transform.PlaneToPlane(Plane.WorldXY, new Plane(new Point3d(MS, 0, 0), Vector3d.XAxis, Vector3d.YAxis))`.

   **Deckel target plane:** Same as Boden but at Z = H-MS.
   → `Transform.Translation(MS, 0, H - MS)`

   **Rueckwand target plane:** back wall, stands vertically at Y=T (back edge), in XZ plane.
   Flat panel: width=innerB along X, height=innerH along Y. Assembled: width along X at offset MS, height along Z at offset MS, positioned at Y=T-MS (or Y=T for eingelegt, but Phase 9.1 placeholder: back face at Y=T, panel thickness goes inward, so origin at Y=T-MS).
   Source: WorldXY. Target: Origin=(MS, T-MS, MS), XAxis=(1,0,0), YAxis=(0,0,1).
   → `Transform.PlaneToPlane(Plane.WorldXY, new Plane(new Point3d(MS, T-MS, MS), new Vector3d(1,0,0), new Vector3d(0,0,1)))`

   Set each panel's `AssembledTransform` and `AssembledOrigin` accordingly.
